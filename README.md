# سامانه رزرو کافه

این پروژه یک وب‌اپلیکیشن رزرو میز برای کافه است که با ASP.NET Core MVC پیاده‌سازی شده است. هدف پروژه، نمایش یک نمونه کاربردی از مدیریت کاربران، محصولات کافه، میزها و رزروها در یک سیستم سمت سرور است.

در این سامانه کاربر می‌تواند محصولات کافه را مشاهده کند، ثبت‌نام و ورود انجام دهد، میزهای قابل رزرو را ببیند و برای تاریخ و ساعت دلخواه خود میز رزرو کند. مدیر سیستم نیز امکان مدیریت کاربران، محصولات، میزها و مشاهده رزروها را دارد.

## امکانات اصلی

- نمایش منوی محصولات کافه همراه با تصویر، توضیح، قیمت و وضعیت موجودی
- ثبت‌نام و ورود کاربران
- تفکیک نقش کاربر عادی و مدیر
- پنل مدیریت برای کاربران مدیر
- مدیریت محصولات شامل ایجاد، ویرایش، مشاهده و حذف
- مدیریت میزهای قابل رزرو شامل قیمت و وضعیت فعال یا غیرفعال
- ثبت رزرو میز توسط کاربر عادی
- نمایش رزروهای هر کاربر به صورت جداگانه
- نمایش همه رزروها برای مدیر
- انتخاب تاریخ رزرو با رابط فارسی و تقویم شمسی
- ذخیره تاریخ رزرو به صورت `DateTime` در پایگاه داده
- جلوگیری از رزرو هم‌زمان یک میز در بازه‌های دارای تداخل

## قانون تداخل رزرو

هر رزرو به مدت ۱ ساعت و ۳۰ دقیقه در نظر گرفته می‌شود. بنابراین اگر یک میز در ساعت ۱۹:۰۰ رزرو شده باشد، همان میز تا ساعت ۲۰:۳۰ برای کاربر دیگری قابل رزرو نیست.

در زمان ثبت رزرو، سیستم بازه انتخاب‌شده را با رزروهای قبلی همان میز مقایسه می‌کند. اگر بازه جدید با رزرو قبلی تداخل داشته باشد، عملیات ثبت انجام نمی‌شود و یک پیام خطا/مودال به کاربر نمایش داده می‌شود که مشخص می‌کند میز تا چه زمانی رزرو است.

## نقش‌های کاربری

### کاربر عادی

کاربر عادی بعد از ورود می‌تواند:

- محصولات کافه را مشاهده کند
- میزهای قابل رزرو را ببیند
- برای یک تاریخ و ساعت مشخص میز رزرو کند
- فقط رزروهای خودش را مشاهده کند

### مدیر

مدیر سیستم می‌تواند:

- همه رزروها را مشاهده کند
- کاربران را مدیریت کند
- محصولات را مدیریت کند
- میزهای رزرو را مدیریت کند
- وضعیت فعال یا غیرفعال بودن میزها را تغییر دهد

## توضیح بخش‌های مهم

### `DataContext`

کلاس `DataContext` ارتباط برنامه با پایگاه داده را برقرار می‌کند و شامل جدول‌های اصلی زیر است:

- `Users`
- `Products`
- `ReserveTables`
- `Reservations`

### `DbInitializer`

این کلاس هنگام اجرای برنامه، در صورت خالی بودن پایگاه داده، تعدادی داده اولیه ایجاد می‌کند؛ مانند:

- کاربر مدیر
- کاربر عادی
- چند محصول نمونه
- چند میز نمونه برای رزرو

### `ReservationTablesController`

این کنترلر مسئول منطق رزرو میز است. مهم‌ترین وظایف آن:

- نمایش میزهای قابل رزرو
- دریافت تاریخ شمسی از فرم
- تبدیل تاریخ شمسی به `DateTime`
- بررسی فعال بودن میز
- بررسی تداخل ۹۰ دقیقه‌ای رزروها
- ثبت رزرو در صورت معتبر بودن اطلاعات

### `JalaliDate`

این کلاس برای کار با تاریخ شمسی استفاده شده است. تاریخ در رابط کاربری به شکل شمسی نمایش داده می‌شود، اما در پایگاه داده به صورت `DateTime` ذخیره می‌شود تا ساختار داده استاندارد باقی بماند.

## نحوه اجرای پروژه

### پیش‌نیازها

برای اجرای پروژه موارد زیر لازم است:

- .NET 8 SDK
- SQL Server LocalDB یا SQL Server
- Visual Studio یا Visual Studio Code

### تنظیم پایگاه داده

رشته اتصال در فایل `appsettings.json` قرار دارد:

```json
"ConnectionStrings": {
  "localSql": "Data Source=(localdb)\\MSSQLLocalDB;Initial Catalog=cafe;Trusted_Connection=True;Integrated Security=true;"
}
```

در صورت نیاز می‌توان نام دیتابیس یا سرور را تغییر داد.

### اجرای Migrationها

برای ساخت پایگاه داده، از ریشه پروژه دستور زیر را اجرا کنید:

```bash
dotnet ef database update
```

### اجرای برنامه

بعد از آماده شدن پایگاه داده:

```bash
dotnet run
```

سپس آدرس نمایش داده‌شده در ترمینال را در مرورگر باز کنید.

## سناریوی نمونه برای تست رزرو

۱. با یک کاربر عادی وارد شوید.  
۲. یک میز را برای ساعت ۱۹:۰۰ رزرو کنید.  
۳. با کاربر دیگری وارد شوید.  
۴. همان میز را در بازه ۱۹:۰۰ تا ۲۰:۳۰ رزرو کنید.  
۵. سیستم باید از ثبت رزرو جلوگیری کند و پیام تداخل زمانی نمایش دهد.  
۶. اگر ساعت خارج از این بازه انتخاب شود، رزرو باید با موفقیت ثبت شود.

## جمع‌بندی

این پروژه یک نمونه کامل از پیاده‌سازی CRUD، احراز هویت ساده، نقش‌های کاربری، ارتباط با پایگاه داده و منطق رزرو در ASP.NET Core MVC است. تمرکز اصلی پروژه روی طراحی یک جریان واقعی برای رزرو میز در کافه، مدیریت داده‌ها توسط مدیر، و نمایش فارسی/راست‌چین برای کاربران فارسی‌زبان بوده است.
